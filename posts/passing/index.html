Programming: what is passing?

February 15, 2019

In Python, so-called primitive types are passed by value.

>>> x = 5
>>> x
5
>>> y = x
>>> y
5
>>> y = 7
>>> y
7
>>> x
5

Other types are passed by reference. This means that the memory location of an object is passed, and modifications (even to a different variable name that’s pointing to the same location) affect that one object.

>>> w = [1,2,3]
>>> w
[1, 2, 3]
>>> z = w
>>> z
[1, 2, 3]
>>> z[1] = 17
>>> z
[1, 17, 3]
>>> w
[1, 17, 3]

Even functions sometimes are passed an object by reference rather than by value. Consider the following code:

# passByRef.py

class Ref(object):
	def __init__(self, name):
		self.name = name

# Note that the function below is NOT part of the class
# Indentation is what determines this
def someFunction(x):
	print x

myObject = Ref("My Object")
print myObject
someFunction(myObject)

If we run this, the result is:

$ python passByRef.py
<__main__.Ref object at 0x7fa30071e750>
<__main__.Ref object at 0x7fa30071e750>

As you can see, the memory location of the object is identical to the memory location of the object the function is using. Use caution!

C++

In general I’m wary of using C++ because it gives so much power to the programmer, and with great power comes the opportunity to inadvertantly do a lot of bad things. But it does let us play with passing by reference in a more explicit manner than Python or Java, so let’s check it out. In the example below, we have three integer variables: x, y, and z, just doing normal integer things. There’s a fourth variable, *w, which is a pointer. C++ lets you explicitly make a variable that points to a certain memory address. In the example, we point *w to the address of y, and we notice that changing the value of *w changes the value of y. After this we point *w to x’s memory address just to show that we can do this too. Notice that in the print statements w shows a memory address and *w shows the integer value stored at that address.

#include <iostream>
using namespace std;
int main()
{
  int x = 5;
  int y = 20;

  cout << "1)   x is " << x << endl;
  cout << "2)   y is " << y << endl;

  int z = x;
  // w is a pointer, and we will point it to y's address:
  int *w = &y;

  cout << "3)   z is " << z << endl;
  cout << "4)   w is " << w << endl;
  cout << "5)  *w is " << *w << endl;

  
  z = 15;
  *w = 42;

  cout << "6)   After changes:" << endl;
  cout << "7)   x is " << x << endl;
  cout << "8)   y is " << y << endl;
  cout << "9)   z is " << z << endl;
  cout << "10)  w is " << w << endl;
  cout << "11) *w is " << *w << endl;

  // Change what w is pointing to
  w = &x;

  cout << "12)  After reassigning w to x's address:" << endl;
  cout << "13)  w is " << w << endl;
  cout << "14) *w is " << *w << endl;
  
  return 0;
}

Running this program gives:

1)   x is 5
2)   y is 20
3)   z is 5
4)   w is 0x7ffd28edfa88
5)  *w is 20
6)   After changes:
7)   x is 5
8)   y is 42
9)   z is 15
10)  w is 0x7ffd28edfa88
11) *w is 42
12)  After reassigning w to x's address:
13)  w is 0x7ffd28edfa8c
14) *w is 5
